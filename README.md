# 深层嵌套文件夹清理工具 (Deep Folder Cleaner)

一个用于删除 Windows 系统中超长路径（6000层以上嵌套）文件夹的工具。解决 Windows 资源管理器、cmd、PowerShell 等常规工具无法删除深层嵌套文件夹的问题。

## 功能特点

- ✅ **超长路径支持**：使用 `\\?\` 长路径前缀，绕过 Windows 260 字符 MAX_PATH 限制
- ✅ **支持分支结构**：栈式后序遍历算法，可处理任意分支结构的嵌套目录
- ✅ **图形化界面**：直观的 GUI 操作，支持浏览选择文件夹
- ✅ **实时进度**：进度条 + 日志输出，随时了解删除进度
- ✅ **可随时停止**：支持中途停止操作
- ✅ **日志保存**：可将操作日志保存为文本文件
- ✅ **安全确认**：删除前弹出确认对话框，防止误操作
- ✅ **单文件 exe**：无需安装 Python，双击即用

## 下载与使用

1. 下载 `DeepFolderCleaner.exe`
2. 双击运行
3. 点击「浏览...」选择要清理的文件夹
4. 点击「开始清理」
5. 在确认对话框中点击「是」开始删除

> **注意**：EXE 版本约 15 MB，已打包所有依赖，无需安装 Python 环境。

## 适用场景

- 游戏模组（MOD）文件夹中恶意/意外生成的深层嵌套目录
- 病毒或恶意软件创建的超长路径文件夹
- 程序 Bug 导致的异常深层目录
- 任何常规方法（资源管理器、cmd rmdir、PowerShell、robocopy）无法删除的嵌套文件夹

## 工作原理

### 1. 长路径支持

Windows API 默认限制路径长度为 260 个字符（MAX_PATH）。通过在路径前添加 `\\?\` 前缀，可以将路径长度扩展到约 32767 个字符，从而访问和操作深层嵌套的目录。

```
普通路径:   C:\very\long\path\...
长路径:    \\?\C:\very\long\path\...
```

### 2. 栈式后序遍历

使用栈（Stack）实现后序遍历，自底向上删除目录：

1. 将根目录压入栈，标记为「未处理」
2. 弹出「未处理」目录：
   - 枚举该目录下所有子项
   - 删除该层级的文件
   - 将该目录重新压入栈，标记为「已处理」
   - 将所有子目录逆序压入栈（标记为「未处理」）
3. 弹出「已处理」目录：
   - 此时所有子目录已被删除
   - 直接删除该空目录
4. 重复直到栈为空

### 3. 删除策略

1.在每个目录层级，先删除所有文件，再处理子目录
2.单个目录删除失败不影响整体进度，错误计数超过阈值才终止
3.遍历删除完成后，尝试用 `shutil.rmtree` 做最终清理

### Q: 删除需要多长时间？

A: 取决于嵌套层数和目录数量。一般来说：
- 1000 层线性嵌套：约 10-30 秒
- 7000 层线性嵌套：约 2-5 分钟
- 分支结构取决于总目录数，每秒约 50-200 个目录

### Q: 为什么不用 `shutil.rmtree`？

A: `shutil.rmtree` 在路径超过 MAX_PATH 限制时会失败。本工具使用 `\\?\` 长路径前缀解决此问题。

### Q: 为什么不用 PowerShell？

A: PowerShell 的 `Remove-Item -Recurse` 在深层嵌套时同样受路径长度限制，且递归深度过大时可能导致栈溢出。

### Q: 删除后文件能恢复吗？

A: **不能**。删除是永久的，不会放入回收站。请谨慎操作，删除前务必备份重要数据。

### Q: 支持 Linux/macOS 吗？

A: 目前仅支持 Windows。Linux 和 macOS 没有 MAX_PATH 限制（或限制很大），常规命令即可处理。

### Q: 为什么 EXE 文件这么大（15MB）？

A: 因为打包了完整的 Python 运行时和 Tcl/Tk GUI 库，所以不需要用户安装 Python。
